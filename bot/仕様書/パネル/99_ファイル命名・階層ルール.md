# ファイル命名・階層統一ルール（日本語版）

## 1. 基本原則
- **言語**: 視認性を高めるため、**日本語ファイル名**を積極的に使用する。
- **目的**: 日本人の開発者がパッと見て何をするファイルか理解できるようにする。

## 2. 命名規則

### ディレクトリ
- **形式**: 日本語名、またはわかりやすい英語名。
- **推奨**: 機能ごとに日本語でディレクトリを切ることを推奨。
- **例**:
    - `管理者パネル/` (OK)
    - `送迎パネル/` (OK)
    - `utils/` (慣習的に英語でも可だが `便利機能/` 等も許容)

### ファイル
- **形式**: `[機能名].js` または `[機能名]_[詳細].js`
- **例**:
    - `管理者パネル.js` (OK)
    - `送迎登録処理.js` (OK)
    - `panel_管理者パネル.js` (既存の形式もOKだが、統一するなら `管理者パネル.js` の方がシンプル)

### システムファイル
- `index.js` や `config.js` など、Node.jsの慣習として英語が一般的なものはそのままで良い。
- ただし、中身が特定のビジネスロジックなら日本語化しても良い。

## 3. ディレクトリ構成ルール

`src/bot` 配下の構成案。

### `command/`
- スラッシュコマンド定義。
- ファイル名: `00_パネル設置.js` のように番号付きや日本語名で管理。

### `handler/`
- 機能ごとに日本語ディレクトリを作成して分類する。

```
handler/
├── index.js                  # ルーティング（ここは英語のままが一般的）
├── 共通処理/
│   └── パネル設置共通.js
├── パネル設置/
│   ├── メイン.js             # 旧 panel_パネル設置パネル.js
│   ├── 埋め込み作成.js       # 旧 buildPanelSetupEmbed.js
│   └── ボタン処理/
│       ├── 管理者ボタン.js
│       ├── 送迎者ボタン.js
│       └── ...
├── 管理者パネル/
│   ├── メイン.js             # 旧 panel_管理者パネル.js
│   └── ...
├── 送迎パネル/
│   ├── メイン.js             # 旧 panel_送迎パネル.js
│   └── ...
├── 利用者パネル/
│   ├── メイン.js             # 旧 panel_利用者パネル.js
│   └── ...
└── 登録関連/
    ├── 送迎者登録.js
    └── 利用者登録.js
```

## 4. リファクタリング対応表

| カテゴリ | 旧ファイル名 | 新ファイル名（案） |
| :--- | :--- | :--- |
| **パネル設置** | `panel_パネル設置パネル.js` | `パネル設置/メイン.js` |
| | `buildPanelSetupEmbed.js` | `パネル設置/埋め込み作成.js` |
| **管理者** | `panel_管理者パネル.js` | `管理者パネル/メイン.js` |
| **送迎者** | `panel_送迎パネル.js` | `送迎パネル/メイン.js` |
| **利用者** | `panel_利用者パネル.js` | `利用者パネル/メイン.js` |

## 5. インポートのルール
- `require('./管理者パネル/メイン')` のように、パスを見れば何をしているか一目瞭然にする。
